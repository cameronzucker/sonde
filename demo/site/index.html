SONDE HF DATA MODEM
ARDOP · SIMULATED CHANNEL EMCOMM · SITREP DELIVERY
▲

Real reference modem · simulated channel · not on-air. Each run drives a real ARDOP modem (ardopcf) end-to-end through a synthetic HF channel on the backend, then plays back what actually decoded. No radio is keyed. ARDOP is a known-good external baseline — the bar Sonde is being built to meet — and is never reimplemented here. There is no link layer, so a frame the channel corrupts is simply lost: watch the image gain holes as conditions worsen.

What you're seeing

  1. Set the channel. Drag SNR and pick a condition. Auto picks the fastest frame type expected to close the link; Manual lets you force one (robust modes need many more frames).
  2. ARDOP runs the real modem. The backend chunks a ~3 KB image across real ARDOP frames, pushes each through a synthetic HF channel, and decodes it — real modulation, real Reed–Solomon FEC, no shortcuts.
  3. Press play. The waterfall sweeps the real transmission, the frame log shows each frame's decode and FEC effort, and the recon photo rebuilds from the frames that survived — with holes where frames were lost.
